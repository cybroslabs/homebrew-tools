## omqctl benchmark

Run performance benchmarks against an OctopusMQ instance

### Synopsis

The 'benchmark' command creates a temporary queue, runs a series of performance
benchmarks (throughput, keyed messages, grouping, deduplication, priorities),
and prints a report. The benchmark queue is always deleted on completion or interruption.

Each scenario runs until at least --message-count messages are processed AND
at least --min-duration has elapsed, whichever is longer.

```
omqctl benchmark [flags]
```

### Options

```
      --batch-size int32        Batch size for pull operations (default 10)
      --concurrency int         Number of concurrent producer/consumer goroutines (default 4)
  -h, --help                    help for benchmark
      --message-count int       Minimum number of messages per scenario (default 10000)
      --message-size int        Payload size in bytes (default 256)
      --min-duration duration   Minimum duration per scenario (e.g. 10s, 1m) (default 10s)
      --priorities uint32       Number of priority levels for the queue (default 3)
```
